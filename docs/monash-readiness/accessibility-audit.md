# Accessibility Audit

## Automated Checks Completed
- Home
- Case library
- Prebrief
- Non-3D encounter path
- Debrief and reflection
- Educator workspace
- Consent and learner-preference controls

## Automated Coverage Notes
- Automated checks focus on accessible names, visible labels, heading presence, button semantics, and key form controls.
- Automation does not claim WCAG certification or replace specialist review.

## Manual Procedures
- Keyboard-only completion
- Screen-reader completion
- Zoom to at least 200%
- High-contrast or forced-colours mode
- Reduced motion preference validation
- Mobile viewport completion
- Error recovery and status messaging
- Transcript navigation
- Long debrief content review
- Educator review tables and export controls

## Manual Checks Still Required
- Full screen-reader narration quality
- Focus order across overlays and educator review forms
- Colour-contrast verification beyond tool-supported coverage
- Cognitive-load review of long debrief pages
- Manual review of chart alternatives and summary comprehension

## Issues Requiring Specialist Review
- Rich simulated consultation surfaces still need human assistive-technology testing.
- Long-form educator analytics views should be reviewed with real pilot data.

## Current Status
- Accessibility automation has been added as a regression guard.
- Manual accessibility review is still required before any institutional pilot claim beyond prototype readiness.
